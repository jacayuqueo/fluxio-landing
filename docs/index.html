Topología de red — algoritmos de grafo

DFS — Enumeración de caminos de flujo

La red se representa como un árbol dirigido. El motor ejecuta DFS desde cada nodo terminal (rejilla) hasta la raíz (HVAC), enumerando todos los caminos de flujo posibles y determinando la sección crítica por máxima caída acumulada.

analisis_flujo.py

BFS inverso — Propagación de caudal acumulado

Procesamiento bottom-up desde hojas hasta raíz: cada nodo terminal aporta su Q_dis definido, y cada nodo de bifurcación (TEE, WYE) acumula la suma de caudales de sus ramas descendentes. El Q total llega al HVAC sin entrada manual.

analisis_flujo.py

Reconocimiento de patrones topológicos

El motor clasifica cada nodo según su contexto en el grafo: un Damper–Transición–Damper con ramal SR5-13 activa una ruta de cálculo diferente a un TEE estándar. La tabla ASHRAE seleccionada depende del patrón, no solo del tipo declarado.

analisis_secuencia.py

Propagación dimensional en cascada

Al modificar la sección transversal de un tramo, el motor recorre aguas abajo y recalcula las relaciones geométricas de cada fitting — una Transición mantiene su razón A₁/A₀ y aplica las nuevas dimensiones sin intervención del usuario.

analisis_secuencia.py
Motor hidráulico

Altshul–Darcy-Weisbach con interpolación ASHRAE multidimensional.

Sin factores de seguridad arbitrarios ni coeficientes genéricos. Cada C₀ se obtiene por interpolación sobre las tablas originales del ASHRAE Duct Fitting Database — las mismas que usa el Handbook para el cálculo manual.

Fórmula de Altshul — Coeficiente de Fricción de Darcy
λ = 0.11 × (ε/Dh + 68/Re)^0.25
Si λ < 0.018 → corrección de Nikuradse: λ = 0.85λ + 0.0028 (régimen hidráulicamente liso)
// Implementado en: Interpolación.py → funpar() · líneas 1237–1243
⚡ Parámetros hidráulicos
Velocidad del flujo
V = Q / A = Q / (W × H)
Q [m³/s] · A [m²] = W × H / 10⁶ — convierte mm² a m²
Diámetro hidráulico
Dh = 4A / P = 4WH / 2(W+H)
P = perímetro húmedo [mm] · Dh [mm]
Número de Reynolds (aproximación aire)
Re = 66.4 × Dh × V
Dh [mm] · V [m/s] → simplifica ρ/μ del aire a T ambiente
Presión dinámica
Pd = ρV² / 2
ρ = 1.2 kg/m³ (corregible por altitud y T_bs)
📐 Pérdidas de presión
Pérdida por fricción — Darcy-Weisbach
ΔP_f = (λ / Dh) × Pd × L
L [mm] → L/1000 [m] · resultado en Pa
Pérdida dinámica — Fittings ASHRAE
ΔP_d = C₀ × Pd
C₀ interpolado de tablas ASHRAE según tipo y geometría
Pérdida total por sección
ΔP_t = ΔP_f + ΔP_d
Suma por tramo, acumulada en la sección crítica
Restricción de diseño
Σ ΔP_t ≤ ESP_dis
Hard constraint — External Static Pressure del equipo RTU
Jerarquía de interpolación — coeficiente C₀
1D
Una variable
El C₀ depende de una única variable independiente (e.g., ángulo de apertura θ). El motor localiza los dos puntos encuadrantes en la tabla y aplica interpolación lineal simple.
CR3_1A · CR9_6 · ER5_3B
ER5_3S · SR5_3S · SR5_5S
SR5_14 · T14_14X · T14_14QS
2D
Dos variables
C₀ = f(x, y) con dos variables independientes (e.g., θ y h/w). El motor localiza la celda encuadrante en la grilla bidimensional y aplica interpolación bilineal sobre los cuatro vértices.
CR3_1 · CR3_6 · CR9_1/3/4
ER3_1 · ER4_1/2 · SR3_1
SR4_1/2 · SR5_3/5/13/15
SR7_2/17 · T14_11F/G/J
3D
Tres variables
C₀ = f(x, y, w) con tres variables (e.g., θ, A₁/A₀, Re). El motor selecciona los dos planos W encuadrantes, ejecuta interpolación bilineal independiente en cada uno y combina linealmente los resultados — interpolación trilineal completa.
SR2_3 — ángulo × A1/Ao × Re
SR2_5 — ángulo × A1/Ao × Re
SR5_1B — Qb/Qc × As/Ac × Ab/Ac
SR5_1S — Qs/Qc × As/Ac × Ab/Ac

🔺 Interpolación 3D Trilineal — interpolar_3d_solo_interpolacion() · Interpolación.py
# Interpolación trilineal: 3 variables (X, Y, W) → valor único
# Aplicado a SR2_3: (ángulo, A1/Ao, Re/1000) → C₀
def interpolar_3d(X, Y, W, tabla):
    # 1. Extraer planos W disponibles de la tabla
    w_planes = agrupar_por_W(tabla)          # {w_val: {y_coords, z_grid}}
    w1, w2 = encontrar_intervalo(W, w_planes) # Planos encuadrantes

    # 2. Interpolación 2D en cada plano
    val_w1 = interpolar_2d(X, Y, tabla_plano(w1)) # Bilineal en plano w1
    val_w2 = interpolar_2d(X, Y, tabla_plano(w2)) # Bilineal en plano w2

    # 3. Interpolación lineal entre planos
    dw = w2 - w1
    return val_w1 + (val_w2 - val_w1) * ((W - w1) / dw) # Resultado

Salidas del análisis hidráulico

📊 Tabla de pérdidas por tramo

Cada sección muestra velocidad, presión dinámica, diámetro hidráulico, Reynolds, coeficiente de fricción λ, pérdida por fricción, C₀ del fitting y pérdida total acumulada — todo en Pa.

✅ Verificación Σ ΔP ≤ ESP_dis

La sección crítica se identifica automáticamente por máxima caída acumulada. El motor compara Σ ΔP contra el External Static Pressure del RTU y reporta el margen disponible o la condición de sobreextensión hidráulica.

📄 Reporte PDF listo para entregar

Un PDF con la tabla completa, la sección crítica marcada y los parámetros del equipo. Generado con un clic — sin copiar nada a Excel ni a Word.

Instalación

Setup en tres pasos. Sin dependencias propietarias.

bash
# 1. Clonar
$ git clone https://github.com/jacayuqueo/fluxio-landing
$ cd hvac-duct-analyzer

# 2. Dependencias base
$ pip install pyqt6 pyvista pyvistaqt numpy vtk

# Opcional — importación IFC
$ pip install ifcopenshell

# Opcional — tessellation STEP precisa
$ pip install gmsh

# 3. Ejecutar
$ python main_controlador.py
1

Clona el repositorio

Necesitas Python 3.10 o más reciente. Funciona en Windows 10/11. Linux y macOS en prueba.

2

Instala las dependencias

El núcleo es PyQt6 + PyVista + NumPy. Si vas a importar modelos BIM agrega ifcopenshell o gmsh según el formato.

3

Ejecuta y empieza a diseñar

Nombre del proyecto → equipo RTU → red de ductos. El cálculo ASHRAE corre mientras construyes la red, sin botón de "calcular".
